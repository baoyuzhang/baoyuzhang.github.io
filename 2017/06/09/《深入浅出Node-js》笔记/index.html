《深入浅出Node.js》笔记

发表于 2017-06-09 | 分类于 Node.js | 阅读次数 | 阅读次数

本文整理我在学习《深入浅出Node.js》过程中的总结与思考，将伴随我的进度持续更新。

一、Node 的特点

单线程的Node，通过回调的方式进行异步编程，达到非阻塞的效果。

1. 异步I/O

— 等待响应期间
同步I/O 不能做任何事，只能等待
异步I/O 继续做其他事

例：读取两个文件的耗时

  • 同步I/O：文件1耗时 + 文件2耗时
  • 异步I/O：最长耗时（文件1，文件2）

2. 事件与回调函数

回调：异步编程的基本方法

每个事件都绑定回调函数，代码的编写顺序与执行顺序无关。

3. 单线程

a. 进程与线程：

  • 进程：分配资源，进程间可通信
    1. 定义：程序在一个数据集合上的一次执行
    2. 组成：程序、数据、进程控制块(PCB)
    3. 特点：
      • 动态：创建进程–执行程序–撤销进程
      • 并发：可同时执行多个进程
    4. 性质：分配存储资源的基本单位
    5. 意义：多进程使多个程序并发执行
  • 线程：独立运行
    1. 性质：分配处理机运行的基本单位
    2. 意义：多线程减少程序并发执行的时间空间开销
